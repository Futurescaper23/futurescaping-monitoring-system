FutureScaping Monitoring System

Sandbar Volume Change Workflow

A practical QGIS prompt sheet for repeating the same sandbar volume-change method across future survey rounds. This is the workflow we used to build the first Area 4 preview in the app.

Core Idea

What stays fixed

The monitored sandbar polygon. Keep the same polygon through time so each comparison is measuring the same sandbody.

What changes

The survey surface. Compare one survey surface against the next one inside the same polygon.

The volume calculation is not based on the aerial image alone. It is based on the difference between two survey surfaces, measured inside a fixed polygon.

Before You Start

Load both survey surfaces

Bring the earlier and later survey surfaces into QGIS.

Load the sandbar polygon

Use the polygon that marks the monitored sandbar or monitoring zone.

Check the basics

Same CRS, same pixel size, same alignment, and as much shared coverage as possible.

Step-by-Step QGIS Workflow

  1. Create or load a shared valid coverage polygon.

    Draw a polygon around the area both survey surfaces genuinely cover and that you trust for analysis.

  2. Clip the earlier survey surface to that shared coverage polygon.

    Use Clip Raster by Mask Layer.

  3. Clip the later survey surface to that same shared coverage polygon.

    Use the exact same mask polygon so both clipped surfaces match.

  4. Subtract the earlier surface from the later surface. 
    "later_survey_surface@1" - "earlier_survey_surface@1"

    Positive values mean the later survey is higher there. Negative values mean it is lower there.

  5. If the subtraction looks stripy or nonsensical, stop and fix alignment first.

    Bad subtraction usually means mismatched grids, mismatched coverage, or no-data problems.

  6. Edit the sandbar polygon if needed.

    Make sure it actually covers the meaningful change zone of the real sandbody you want to measure.

  7. Clip the difference raster to the sandbar polygon.

    That gives you the change surface only inside the monitored sandbar.

  8. Find the raster cell size.

    In Layer Properties, note the pixel width and height.

  9. Calculate cell area. 
    cell area = pixel width × pixel height
  10. Create three volume rasters.

    One for material added, one for material removed, and one for overall balance.

  11. Read the totals from those three rasters.

    Those three totals are the headline numbers that go into the app.

Raster Calculator Formulas

Once you have the clipped difference raster and the cell area, use the following formulas.

Difference raster

"later_clipped_surface@1" - "earlier_clipped_surface@1"

Cell area

pixel width × pixel height

Material added

("diff_clipped@1" > 0) * "diff_clipped@1" * CELL_AREA

Material removed

("diff_clipped@1" < 0) * abs("diff_clipped@1") * CELL_AREA

Overall balance

"diff_clipped@1" * CELL_AREA

Interpretation

Added and removed are positive headline totals. Overall balance can be positive or negative.

How To Style The Change Preview PNG

Base

Use the ortho as the base image.

Overlay

Place the clipped difference raster over the ortho with partial transparency.

Outline

Show the sandbar polygon outline only. No solid fill.

Colour range

Use a diverging ramp with red/orange for lowering, pale centre for little change, and green/blue for build-up.

Manual range

Set the raster min/max manually. A good starting point is -0.25 to 0.25 metres.

Export

Export one PNG against the earlier ortho and one against the later ortho if you want both contexts in the app.

What Goes Into The App

Preview images

The change overlay PNG against the earlier ortho and the later ortho.

Headline figures

Material added, material removed, and overall balance.

Simple explanation

A short note explaining what changed in plain language.

Confidence

A quick confidence label such as Low, Medium, or High.

Troubleshooting

Blank or ridiculous difference raster

Usually a grid alignment, no-data, or extent problem. Recheck clipping and alignment first.

Clip-by-mask error

If QGIS says a cutline feature has no geometry, export only the one real polygon as a clean new layer and use that as the mask.

Too much white space in the preview

Zoom tighter before exporting the PNG so the monitored sandbar is the main subject.

Change not showing clearly

Tighten the pseudocolour min/max range and reduce the raster opacity slightly until the ortho and change both read well.

Quick Checklist

  • Same CRS, same pixel size, same alignment.
  • Clip both survey surfaces to the same valid shared coverage.
  • Subtract later minus earlier.
  • Clip the difference raster to the monitored sandbar polygon.
  • Use the real cell area in the raster calculator formulas.
  • Extract material added, material removed, and overall balance.
  • Export a clean preview PNG and add the headline figures to the app.

Recommended habit: reuse the same polygon names, file names, and output labels every time so each new survey round feels like repeating a system rather than reinventing a method.
