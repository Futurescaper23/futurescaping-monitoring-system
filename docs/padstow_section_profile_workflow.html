Section Profile Workflow

A repeatable QGIS workflow for generating and cleaning section-profile CSVs for the monitoring app. This guide follows the same logic we used to get the live section comparison view working.

Core Principle

What stays fixed

The section lines. One fixed set of section lines per area.

What changes each survey

The sampled profile CSV. One CSV per survey round per area.

Step-by-Step QGIS Workflow

  1. Load the correct survey surface.

    Bring the survey surface for the round you want to sample into QGIS.

  2. Load the fixed section line layer.

    Do not redraw the lines for each survey if you want direct comparison through time.

  3. Create points along the geometry.

    Use Points Along Geometry on the section lines.

    Distance: 1 m
Start offset: 0
End offset: 0
  4. Sample the survey surface.

    Use Sample Raster Values on the points layer and the current survey surface.

  5. Check the sampled field.

    The sampled values usually land in a field like SRV1.

  6. Convert no-data to 0 in the cleaned height field.

    This is the approach we used for the app so the section can flatline over no-data stretches.

    if(
    "SRV1" IS NULL,
    0,
    round("SRV1", 2)
)
  7. Make sure the distance field is populated.

    If needed, create or update distance_m from the original distance field.

    "distance"
  8. Export the table as CSV.

    Keep the key fields:

    section_id
label
sort_order
height_m
distance_m
  9. Save one CSV per survey per area.

    Use the same filename each time, but place it in the relevant survey folder.

    survey-data/padstow-estuary/2026-03-22/area4/section_profiles.csv
survey-data/padstow-estuary/2026-04-18/area4/section_profiles.csv

How The App Uses It

Single survey view

The app shows one profile line for the selected survey round.

Overlay view

When multiple survey rounds are ticked, the app overlays the lines in different colours.

Map tracking

The red dot follows the fixed section line while you move across the profile graph.

Zero stretches

Zero/no-data stretches are shown differently from the measured section surface.

Common Checks

  • Make sure the section line IDs match the CSV IDs.
  • Keep the section line geometry fixed between survey rounds.
  • Check that height_m has real values in the middle of the section, not just zeros.
  • Confirm the correct survey surface was used in Sample Raster Values.
  • Use the same rotated export frame for section-line PNGs if the app is using calibrated display positions.
